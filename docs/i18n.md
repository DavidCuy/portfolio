# Internacionalización (i18n)

El sitio soporta dos idiomas con `@nuxtjs/i18n`.

---

## Configuración

```ts
// nuxt.config.ts
i18n: {
  defaultLocale: 'es',
  strategy: 'prefix_except_default',
  locales: [
    { code: 'en', file: 'en.json' },
    { code: 'es', file: 'es.json' }
  ],
  langDir: 'locales/'
}
```

- **Español** (`es`): idioma por defecto, sin prefijo → `/blog`
- **Inglés** (`en`): con prefijo → `/en/blog`

---

## Traducciones de UI

Textos de interfaz (navbar, botones, labels) en:
- `app/locales/es.json`
- `app/locales/en.json`

Uso en componentes:
```ts
const { t, locale } = useI18n()
t('topbar.home')   // → "Inicio" | "Home"
```

---

## Contenido localizado en YAML

Los campos del contenido `.yml` que tienen traducción usan la estructura:
```yaml
title:
  es: "Título en español"
  en: "Title in English"
```

Resolver con `getLocalized`:
```ts
import { getLocalized } from '~/utils/getLocalized'
const { locale } = useI18n()

// En computed:
const title = computed(() => getLocalized(page.value?.title, locale.value))
```

`getLocalized` (`app/utils/getLocalized.ts`):
- Si el campo es `string` → lo devuelve directamente (sin traducción)
- Si es `{ es, en }` → devuelve el valor del locale activo, o el primero disponible como fallback
- Si es `null/undefined` → devuelve `''`

---

## Navegación con i18n

`useNavLinks()` en `app/utils/links.ts` genera los links con el prefijo correcto:
```ts
const prefix = locale.value === 'en' ? '' : `/${locale.value}`
// Español (default): prefix = ''     → /projects
// Inglés:            prefix = ''     → /en/projects  (nuxt/i18n lo añade)
```

> Nota: el Blog no está en el navbar actualmente. Si se añade, seguir el mismo patrón.

---

## Sanity y i18n

El blog de Sanity **no tiene soporte multiidioma** en el schema actual. Todos los posts están en el idioma en que se escriban. Si en el futuro se necesita localización de posts, habría que agregar campos `title_en`, `body_en`, etc. al schema de Sanity o usar el plugin oficial de i18n de Sanity.
